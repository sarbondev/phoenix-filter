# Prestige Filter — Design Brief

**Раздел:** Автомобильная фильтрация для спецтехники
**Дата:** Май 2026
**Технологии:** Next.js 15 (App Router), Tailwind CSS v4, Framer Motion, React 19
**Языки интерфейса:** Русский (основной), Узбекский (Latin), Английский, Қазақша

---

## 1. Контекст проекта

Prestige Filter — B2B/B2C магазин фильтров. Текущий сайт уже покрывает **легковые автомобили** (каталог Phoenix ~3 300 SKU: FIAT, Hyundai, Opel и т.д.).

Сейчас добавляется **новый, отдельный раздел** — **«Автомобильная фильтрация для спецтехники»**. Это другой сегмент:

- **Клиент:** автопарк, карьеры, стройплощадки, агрохолдинги, муниципалитеты.
- **Техника:** экскаваторы, погрузчики, бульдозеры, самосвалы, грейдеры, катки, краны, бетоносмесители, дорожная, сельхоз, коммунальная техника, генераторы.
- **Бренды:** Donaldson, Parker, Fleetguard, MANN, HYDAC, WIX, SF Filter, Baldwin, Sakura, Hengst.
- **Главный сценарий («killer flow»):** клиент вводит OEM-номер от своей машины (например `CAT 1R-1808`) → мгновенно видит аналог Donaldson P551808 → «Запросить цену» или WhatsApp.

Раздел будет жить параллельно с легковой частью. Архитектурно это **отдельная «direction»** (категория верхнего уровня) — поэтому нужна **самостоятельная навигация, hero, оформление и набор сценариев**.

## 2. Бренд и визуальный язык

### Цветовая палитра (уже зафиксирована в коде, менять нельзя)

| Токен | HEX | Применение |
|---|---|---|
| `--color-brand` | `#00205B` | Глубокий тёмно-синий. Основной бренд-цвет, CTA-кнопки, ссылки, заголовки-акценты. |
| `--color-brand-soft` | `#e7ecf4` | Светлый фон для блоков, badges, hover-зоны. |
| `--color-brand-strong` | `#00112e` | Фон тёмных секций (hero). |
| `--color-ink` | `#0f1729` | Почти-чёрный. Тексты, тёмные карточки. |
| `--color-highlight` | `#ffb800` | Янтарный/золотой. Только для главных CTA («Подобрать фильтр», «Получить цену»). Не использовать как фон большого блока. |
| `--color-accent` | `#ef2531` | Красный. Срочность («Заказать сейчас»), индикатор «Нет в наличии». Использовать сдержанно. |
| `--color-surface` | `#f6f7fb` | Серый фон секций (когда нужно отделить от белого). |
| `--color-success` | `#10b981` | «В наличии», подтверждения. |
| `--color-warning` | `#f59e0b` | «Под заказ». |

**Темнее / светлее версии всех тонов — есть в `globals.css` (`-hover`, `-active`, `-soft`).**

### Типографика
- **Headings:** жирный, плотный трекинг (`letter-spacing: -0.025em`).
- **Body:** обычный sans, чуть сжатый (`letter-spacing: -0.011em`).
- **Цифры (OEM, размеры, артикулы)** — рекомендуется monospace или tabular-nums, чтобы колонки в таблицах выровнялись.
- В коде используется системный шрифт; в дизайне допускается Inter / Manrope / IBM Plex Sans.

### Радиусы и тени
- Радиусы: `xs=4px`, `sm=6px`, `md=10px`, `lg=14px`, `xl=18px`, `2xl=24px`. Карточки — `lg`/`xl`. Кнопки — `md`/`lg`.
- Тени: тонкие, без сильного блюра. Базовая `shadow-md` (`0 4px 12px rgba(15,23,41,0.06)`), для подъёма — `shadow-lg`. Брендовая `shadow-brand` (синий glow) — только для главных CTA.

### Настроение
- **Серьёзный, инженерный, надёжный.** Это не глянцевый e-commerce — это B2B для технических специалистов.
- Фотографии — настоящая техника в работе (карьер, стройка, поле), фильтры на чёрном фоне с резким светом.
- Иллюстрации/иконки — линейные, единый stroke-weight (1.5–2px).
- Минимум градиентов. Если градиент — диагональный, brand → ink (как в существующем `.bg-diagonal-blue-ink`).
- Никаких пастельных тонов, скруглённых детских иллюстраций, эмодзи в продакшен-UI.

### Текущие референсы внутри проекта (для согласованности)
- `client/src/app/globals.css` — все токены, утилиты `.section-title`, `.bg-diagonal-blue-ink`.
- `client/src/widgets/hero/Hero.tsx` — текущий hero-карусель направлений.
- `client/src/widgets/hero/BrandsSection.tsx` — лого-сетка брендов.
- `client/src/entities/product/ProductCard.tsx` — карточка товара.

## 4. Главная раздела `/spetstexnika`

Длинная одностраничная композиция. Порядок секций — критичен (по убыванию приоритета конверсии).

### 4.1 Hero
- **Тёмный** фон (фото техники в работе + brand-strong overlay).
- Заголовок (H1): **«Автомобильная фильтрация для спецтехники»**.
- Подзаголовок: «Надёжные фильтры для защиты двигателей, гидравлических систем, топлива и кабины оператора в тяжёлых условиях эксплуатации.»
- 3 CTA-кнопки в один ряд:
  1. **«Подобрать фильтр»** — primary, цвет `--color-highlight`, ведёт на `/filter-search`.
  2. **«Скачать каталог»** — secondary, белая обводка, скачивает PDF.
  3. **«Получить консультацию»** — ghost / link-style, открывает чат-виджет или скроллит к форме.
- Справа от текста (на desktop): крупное фото фильтра / экскаватора, с лёгким Ken Burns эффектом.
- Над всем — узкая полоса 6 преимуществ с иконками: Защита оборудования · Высокая эффективность · Долговечность · Экономия затрат · OEM качество · Быстрая поставка.

### 4.2 Spetstexnika Cards (Блок «По типу техники»)
- Заголовок секции: **«Подбор по типу спецтехники»**.
- Сетка 4×3 (desktop) / 2×6 (tablet) / 2×6 или горизонтальный скролл (mobile) — 12 карточек.
- Карточка: фото техники (сверху, ~16:10), название снизу, при hover — лёгкое поднятие + появляется кнопка-стрелка.
- Клик → страница `/spetstexnika/equipment/excavators` (фильтры, подходящие этой технике).

### 4.3 Filter Categories (9 категорий)
- Заголовок: **«Категории фильтров»**.
- Сетка 3×3 (desktop) / 2×5 (tablet) / 1× в две колонки (mobile).
- Карточка крупнее, чем у техники: иконка (линейная, 48–64px), название, краткое описание (1 строка), линк-стрелка.
- При hover — карточка получает рамку `--color-brand` и приподнимается.

### 4.4 Killer Flow Block — «Найди свой фильтр за 30 секунд»
- Самая важная секция. **Должна быть выше остальных по визуальному весу.**
- Большое поле поиска (как Google) + табы: «OEM номер», «Аналог», «Размер», «Техника», «Фото».
- Под полем — 3-step explainer: 1) Введи номер → 2) Получи аналог → 3) Заказывай.
- Бэкграунд — `--color-surface` или диагональ brand→ink.
- Поле фокусом по умолчанию, placeholder: «Введите OEM номер, артикул, бренд или модель техники».

### 4.5 Применение фильтров
- 4 крупные карточки в ряд (desktop) / 2×2 (tablet) / стек (mobile):
  - **Защита двигателя** (воздух · масло · топливо)
  - **Защита гидросистемы** (насосы · гидромоторы · клапаны)
  - **Защита топливной системы** (форсунки · насос высокого давления)
  - **Чистый воздух в кабине** (HVAC · кондиционирование)
- Карточка: иконка → заголовок → bullets → линк «Подробнее».

### 4.6 Популярные бренды
- Серая или белая секция, заголовок «Работаем с лидерами отрасли».
- Сетка из 10 логотипов в greyscale, при hover — цветной. По клику → отфильтрованный каталог по этому бренду.
- Бренды: Donaldson, Parker, Fleetguard, MANN, HYDAC, WIX, SF Filter, Baldwin, Sakura, Hengst.

### 4.7 Фильтры для тяжёлых условий
- Тёмная секция (`--color-ink`).
- Слева — иллюстрация / фото пыльного карьера. Справа — текст.
- Заголовок: «Работаем там, где другие сдаются».
- Список проблем (карьерная пыль, песок, высокая влажность, жара, вибрация) и решений (heavy-duty air filters, nanofiber, radial seal, reinforced hydraulic) — двумя колонками.
- CTA «Подобрать фильтр для тяжёлых условий».

### 4.8 Подбор фильтра (короткая форма)
- Светлый блок, центрированный, max-width ~720px.
- Поля: марка техники → модель → год → двигатель → тип фильтра → OEM номер (опц.) → телефон.
- Дополнительно: «Загрузить фото фильтра» (drag-and-drop).
- Кнопка «Получить подбор» — `--color-highlight`.
- Disclaimer ниже мелким шрифтом: «Менеджер свяжется в течение 30 минут».

### 4.9 SEO / FAQ
- Аккордеон с 5–8 вопросами:
  - Чем heavy-duty фильтр отличается от обычного?
  - Как часто менять воздушный фильтр на экскаваторе?
  - Можно ли использовать аналог вместо оригинала?
  - Что такое OEM номер и где его найти?
  - и т.п.

### 4.10 CTA + контакты (footer-of-section)
- Полоса с заголовком «Не нашли нужный фильтр?», 2 CTA: «Отправить OEM номер», «WhatsApp».
- Контакты: телефон, адрес, часы работы.

## 6. Страница `/filter-search` — мастер-поиск

Самостоятельная страница. **Дизайн-приоритет #1.** Это и есть «продукт» сайта.

### Структура
1. **Hero (короткий)**
   - Заголовок: «Поиск и подбор фильтров по OEM, аналогам и размерам».
   - Подзаголовок: «Найдите нужный фильтр по OEM номеру, аналогу, размеру или модели техники. Если точного фильтра нет, мы подберём качественный аналог по параметрам.»

2. **Search UI (главный элемент)**
   - Гигантское поле ввода (height ~64px, font 18–20px). Placeholder: «Введите OEM номер, артикул, бренд, модель техники или размер».
   - **Tabs** над полем — 5 вкладок:
     - 🔢 По OEM
     - 🔁 По аналогу
     - 📏 По размеру
     - 🚜 По технике
     - 📷 По фото
   - Активный tab — `--color-brand` подчёркивание, неактивный — `--color-muted`.

3. **Tab «По OEM»**
   - Одно поле + кнопка «Найти».
   - Под полем — подсказка-пример: «Например: CAT 1R-1808, Fleetguard FF5320».
   - Система должна понимать варианты: `1R-1808`, `1R1808`, `1R 1808`, `CAT 1R-1808`, `Caterpillar 1R1808` — все одинаково.

4. **Tab «По аналогу»**
   - Поле + дропдаун выбора бренда (Donaldson, Fleetguard, MANN, Parker, HYDAC, WIX, Sakura, Baldwin).

5. **Tab «По размеру»**
   - Форма 2×3 grid:
     - Тип фильтра (select)
     - Высота, мм (input + допуск ±)
     - Наружный диаметр, мм (input + допуск ±)
     - Внутренний диаметр, мм (input)
     - Резьба (select)
     - Тип уплотнения (select)
   - Кнопка «Найти подходящие».

6. **Tab «По технике»**
   - 4 поля: Бренд техники (select) → Модель (зависимый select) → Двигатель → Год выпуска.

7. **Tab «По фото»**
   - Drag-and-drop зона (большая, пунктирная рамка).
   - Иконка камеры, текст «Загрузите фото фильтра — менеджер подберёт по нему».
   - Поле телефона.
   - Кнопка «Отправить запрос».

8. **Результаты поиска** (ниже UI поиска, после клика «Найти»):
   - Карточка результата (см. раздел 7).
   - Если ничего не найдено — empty state с предложением загрузить фото / связаться с менеджером.

## 7. Карточка фильтра (товара)

Используется и в каталоге, и в результатах поиска. Должна вмещать много данных без визуального шума.

### Layout (список / сетка)
- **Сетка (3 в ряд):** 320×420 примерно.
- **Список (search results):** горизонтальная, 100% × 200.

### Обязательные элементы
1. **Фото** (с серым фоном, по центру).
2. **Бейджи поверх фото** (top-left): «В наличии» / «Под заказ» / «Hit», «Новинка».
3. **Бренд + part_number** — крупно, как заголовок (например `Donaldson P551808`).
4. **Тип фильтра** — мелким, серым (например «Масляный фильтр»).
5. **OEM:** `CAT 1R-1808` — с лейблом «OEM:».
6. **Аналоги:** `Fleetguard LF691A`, `Baldwin B99`, `WIX 51791` — chips, max 3, остальные «+5 ещё».
7. **Размер:** `Ø наруж. 118 мм / высота 260 мм / резьба 1-1/8-16` — одной строкой, monospace.
8. **Применение:** `CAT, Komatsu, генераторы, спецтехника` — одной строкой.
9. **Цена:** «По запросу» (бейдж в `--color-brand-soft`) или фактическая, если есть.
10. **Действия** (на ховере или всегда):
    - Главная: **«Подробнее»** (primary).
    - Вторичная: **«Запросить цену»** (ghost).
    - Утилитарные иконки: «Получить аналог», WhatsApp, Wishlist.

## 11. Состояния и edge cases

Для каждой ключевой страницы дизайнеру нужно нарисовать:

1. **Loading** — скелетоны под структуру (не спиннер по центру).

## 15. Не делать (anti-patterns)

- ❌ Не использовать эмодзи в продакшен-UI (только в этом брифе).
- ❌ Не делать яркие градиенты (только сдержанная диагональ brand→ink).
- ❌ Не делать круглые «дружелюбные» иллюстрации в стиле SaaS-стартапов. Это B2B спецтехника.
- ❌ Не дублировать существующий легковой hero — спецтехника **другой раздел, другая визуальная подача** (темнее, грубее, ближе к индустрии).
- ❌ Не прятать главный CTA «Подобрать фильтр» — он должен быть видим на каждом экране (sticky bar на mobile, hero + поиск + footer на desktop).
- ❌ Не использовать словарь «корзина / купить» в этом разделе — здесь сценарий «запросить цену / получить подбор». Корзина для легковой части.
